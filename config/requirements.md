# requirements.md

## アプリ概要
- アプリ名: 農業日報管理アプリ
- 概要: 農場作業者がロット単位で日報を作成・閲覧できるWebアプリ。
        作業者の入力内容をもとにAIが日報要約を自動生成する。

## デザイン
- Claude Designのハンドオフバンドルを参照すること
- ハンドオフ後のバックエンド繋ぎ込みはimplement.mdで別途指示すること

## 画面 / 基本機能

### ロット一覧画面（/）
- 機能: ロットの一覧表示。1ロット1カードとして表示。
- 表示項目: ロットID・農場名・棟番号・栽培品目・栽培品種・定植本数
- 操作: 「日報を書く」ボタン → 日報作成画面へ遷移
        「日報を見る」ボタン → 日報閲覧画面へ遷移
- 備考: デモ用ロットを3件表示すること
        DEMO_LOT1: KAISHI農場 / 1号棟 / トマト / シンディー / 540本
        DEMO_LOT2: KAISHI農場 / 2号棟 / メロン / アールス / 320本
        DEMO_LOT3: KAISHI農場 / 3号棟 / トマト / フルティカ / 480本

### 日報作成画面（/lots/{lot_id}/report/new）
- 機能: ロット単位で日報を作成する
- 表示項目: ロット情報・日付・植物の状態・病害虫の状況・写真・コメント
- 操作: 植物の状態をスライダーで入力（悪〜良）
        病害虫の状況をスライダーで入力（悪〜良）
        写真アップロード
        コメント入力（200文字以内）
        保存ボタン押下でAI要約生成アニメーションを表示
        完了後に日報閲覧画面へ遷移
- 備考: 保存時に「AIが日報を要約中...」のローディング演出を表示すること
        タイピングアニメーションでAIまとめを表示すること

### 日報閲覧画面（/lots/{lot_id}/report/{report_id}）
- 機能: 作成済み日報の閲覧
- 表示項目: ロット情報・日付・センサーデータ・作業者入力内容・AIまとめ
- 操作: 「ロット一覧へ」ボタン → ロット一覧画面へ遷移
- 備考: センサーデータ（温度・湿度）はダミーデータを固定で表示すること
        AIまとめはsrc/utils/demo_data.pyから固定テキストを返すこと

### ロット登録・編集ダイアログ（src/templates/components/lot_registration_dialog.html）
- 機能: ロットの新規登録・編集・削除
- 表示項目:
  - ロットID（テキストボックス・編集時は変更不可）
  - 農場名（テキストボックス）
  - 棟番号（テキストボックス）
  - 栽培品目（テキストボックス）
  - 栽培品種（テキストボックス・Nullable）
  - 定植本数（テキストボックス・Nullable）
- 操作:
  - 新規登録時: 「登録」ボタン・「キャンセル」ボタン
  - 編集時: 「保存」ボタン・「削除」ボタン・「キャンセル」ボタン
- 備考:
  - 新規登録・編集でダイアログを共通化すること
  - 編集時は登録済みデータを各項目に初期表示すること
  - 編集時はロットIDを変更不可（readonlyに）すること
  - 削除時は該当ロットに紐づくReportsと画像ファイルも削除すること
  - ダイアログの表示はHTMXでcomponents/から取得しinnerHTMLで差し込むこと
  - 新規登録時: GET /lots/dialog/new → 空のダイアログを返す
  - 編集時: GET /lots/{lot_id}/dialog/edit → lot_idでDBからデータを取得し
    登録済みデータを各項目に埋め込んだダイアログを返す
  - ダイアログの開閉はAlpine.jsで管理すること
  - JSON埋め込みはscriptタグ + windowオブジェクト経由で行うこと
    （x-data属性内にJSONを直書きしないこと）

## 非機能要件
- パフォーマンス: アニメーションはスムーズに動作すること
- セキュリティ: 認証不要（デモ版のため）
- 対応デバイス: PCブラウザ専用

## ビジネスロジック
- 1ロットにつき1日1レポートとすること
- 日報作成画面アクセス時に当日レポートの存在を確認すること
  - 存在する場合: 登録済み内容を表示すること
  - 存在しない場合: 新規作成フォームを表示すること
- 現在日時の取得はsrc/utils/datetime_utils.pyの関数を使用すること
- 写真はsrc/imgs/に保存し{date}_{ulid}形式でファイル名を生成すること
- 写真は最大3枚までアップロード可能とすること
- 登録済みの写真は削除できること
  - 削除時はsrc/imgs/から画像ファイルも削除すること
  - Reportsテーブルのphoto_pathsからも該当パスを削除すること
- temperature・humidityはレポート保存時にランダム値を生成して保存すること
  - temperature: 20〜30の範囲でランダム生成（小数点1桁）
  - humidity: 50〜70の範囲でランダム生成（小数点1桁）
  - 生成処理はsrc/utils/demo_data.pyに定義すること

### ロット登録
- ロット一覧画面の右下に「+」ボタンを表示すること
- 「+」ボタン押下でロット登録ダイアログを表示すること
- ダイアログの構成
  - 各項目の入力テキストボックス
    （ロットID・農場名・棟番号・栽培品目・栽培品種・定植本数）
  - 「登録」ボタン・「キャンセル」ボタン

### ロット編集・削除
- 各ロットカードに「編集」ボタンを表示すること
- 「編集」ボタン押下でロット編集ダイアログを表示すること
- ダイアログはlot_registration_dialog.htmlを流用すること
- ダイアログの構成
  - 登録済みロットデータを各項目に表示すること
  - 「保存」ボタン・「削除」ボタン・「キャンセル」ボタン
- 削除時は該当ロットのReportsも合わせて削除すること
  - Reportsに紐づく写真ファイル（src/imgs/）も削除すること

## データ構造

### Lotsテーブル（src/models/lot.py）
- id: String（例: DEMO_LOT1）主キー
- farm_name: String（農場名 例: KAISHI農場）
- house_number: String（棟番号 例: 1号棟）
- crop_type: String（栽培品目 例: トマト）
- crop_variety: String・Nullable（栽培品種 例: シンディー）
- plant_count: Integer・Nullable（定植本数 例: 540）

### Reportsテーブル（src/models/report.py）
- id: String（ULID）主キー
- lot_id: String（外部キー → Lots.id）
- date: String（日付 例: 2026-06-15）
- plant_condition: Integer（植物の状態 1〜5）
- pest_condition: Integer（病害虫の状況 1〜5）
- photo_paths: Text・Nullable（写真パスのJSON配列 最大3枚）
- comment: Text・Nullable（コメント 200文字以内）
- ai_summary: Text・Nullable（AIまとめ）
- temperature: Float（気温 20〜30のランダム値）
- humidity: Float（湿度 50〜70のランダム値）
- created_at: DateTime（作成日時・src/utils/datetime_utils.pyで取得）

## 制約条件
- uvicornでローカル起動すること（デプロイ不要）
- DBはSQLiteを使用すること
- AI要約はダミーテキストで代替すること（src/utils/demo_data.pyに集約）
- センサーデータはダミーデータを固定で表示すること
- デモ映えするアニメーションを積極的に取り入れること
  - 画面遷移時のフェードイン
  - AI要約生成時のタイピングアニメーション
  - 保存完了時のトースト通知
